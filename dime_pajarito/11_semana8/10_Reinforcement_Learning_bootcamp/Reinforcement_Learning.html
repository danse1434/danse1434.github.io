What is reinforcement learning?¶

In your daily life, you are constantly faced with information (i.e. states of your environment) and are required to make decisions (i.e. take actions). The actions you take are meant to move you closer to your ultimate goal in life (whatever that may be); moreover, these actions in turn affect your environment and thus the subsequent information you might encounter.

The key idea behind reinforcement learning (RL) is to use feedback from the environment in the form of rewards or punishments to train an agent for a certain task. Similar to what a child experiences while it is growing up, we want to take a "trial-and-error" approach with the enviroment and learn what our optimal decisions (i.e. policies) should be.

Sequential decision-making¶

In RL, the agent generates its own training data by interacting with the world. The agent must learn the consequences of their own actions by trial-and-error, rather than being told the correct action at each step. Let's introduce this evaluative aspect of RL with an example:

Suppose that this woman is choosing her favorite ice cream. She starts by selecting a ramdon desert and then rates it with an emoji. After a while, she begins to prefer one type of ice cream, and either only eats that ice cream type from now on, or continues with the random selection and exploring her options. If she only eats one ice cream type, then she wouldn't be able to collect data on the other options. Perhaps one of the other ice creams is better, or could be to her in the future.

This example is a case of the $k$-armed bandit problem. In this problem, we have an agent that chooses between $k$ different actions, and receives a reward acording to the action it chooses. In our examples the agent is the woman, and the $k$ different actions are the different types of ice cream she could order. Each action or ice cream type choice will generate a different reward.

Action-values¶

For our agent to decide which action is optimal or best, we most know the reward from taking that action. We call this mapping the action-value function:

\begin{equation*} \large Q_*(a) \doteq \mathbb{E}[R_t|A_t=a] \forall a \in {1,\dots,k} \end{equation*}
\begin{equation*} \large = \sum_r p(r|a)\times r \end{equation*}

The optimal action-value function is defined as the expected reward when taking action $a_i$ for each possible action $i\in\{1,\dots,k\}$. The goal of our agent is to maximize the expected reward:

\begin{equation*} \DeclareMathOperator*{\argmax}{arg\,max} \large \argmax\limits_{a} Q_*(a) \end{equation*}

Sample-average method¶

However, we don't actually know this function $Q_*$! It is unknown to our agent since at the beginning we don't have any information. One possible way of estimating the action-value function is:

\begin{equation*} \large Q(a) \doteq \frac{\text{sum of rewards when $a$ is taken before time $t$}}{\text{Number of times we took $a$ before time $t$}} \end{equation*}
\begin{equation*} \large = \frac{\sum_{i=1}^{t-1}R_i}{t-1} \end{equation*}

Let's assume that our agent receives a reward of 1 if she likes the ice cream she tried that particular time, and 0 otherwise. What would our action-value function estimation look like for 6 time steps?

As our agent takes more actions, our estimates will get closer and closer to the real values of $Q_*$. Generalizing the calculation above, we can keep track of the sample average of our rewards as follows:

\begin{equation*} \large Q_{n+1}= \frac{1}{n}\sum_{i=1}^{n}R_i \end{equation*}

We can pull the current reward out of the sum:

\begin{equation*} \large = \frac{1}{n}(R_n + \sum_{i=1}^{n-1}R_i) \end{equation*}

We write our next value estimate in terms of the previous value estimate - for this, we multiply and divide by $n - 1$:

\begin{equation*} \large = \frac{1}{n}(R_n + (n-1) \frac{1}{n-1}\sum_{i=1}^{n-1}R_i) \end{equation*}

The last term is by definition our current value estimate:

\begin{equation*} \large = \frac{1}{n}(R_n + (n-1) Q_n) \end{equation*}

With a little bit of algebra we end up with a recursive way of updating our action-value function; this is formally known as the incremental update rule:

\begin{equation*} \large Q_{n+1}= Q_n + \frac{1}{n}(R_n - Q_n) \end{equation*}
\begin{equation*} \large \text{New estimate}=\text{Old estimate}+\text{Step size}[\text{Target - Old estimate}] \end{equation*}

Action selection¶

In reality, our agent woudn't choose an action randomly. Rather, it would choose what it believes is the best ice cream acording to its experience up to that point. This method of choosing actions is known as the greedy action selection approach. If the agent only chooses the greedy action, this would be considered an exploitation approach. However, it could also choose non-greedy actions with a certain probability $\epsilon$ to get more information on the other action-values - this is known as a exploration approach. A combination of both is formaly known as an $\epsilon$-greedy method.

Solving the $k$-armed bandit problem¶

Greedy agent¶

Let's first implement an agent that takes a purely greedy approach.

First we are going to implement the argmax function, which takes in a list of action values and returns an action with the highest value. Why are we implementing our own function instead of using the argmax function that numpy uses? Well, numpy's argmax function returns the first instance of the highest value. We do not want that to happen as it biases the agent to choose a specific action in the case of ties. Instead we want to break ties between the highest values randomly:

In [4]:
def our_argmax(q_values):
    """
    Imput: list of q_values 
    Returns: Index of the item with the highest value. Breaks ties randomly.
    """
    value = float("-inf")
    breaks = []
    
    for i in range(len(q_values)):
        if q_values[i]>value:
            value=q_values[i]
            breaks = []
        if q_values[i]==value:
            breaks.append(i)
    return np.random.choice(breaks)

Now we introduce the first part of an RL-Glue agent that you will implement. Here we are going to create a GreedyAgent and implement the agent_step() method. This method gets called each time the agent takes a step. The method has to return the action selected by the agent. This method also ensures the agent’s estimates are updated based on the signals it gets from the environment:

In [5]:
class GreedyAgent(main_agent.Agent):
    def agent_step(self, reward, observation):
        """
        The function agent_stpe takes in a reward and observation and 
        returns the action the agent chooses at that time step.
        
        Imput
        reward: the reward the agent gets from the environment after the last action.
        observation: The state the agents obverves .
        Returns: The action chosen by the agent at the current time step.
        """
        #Number of times we visited each arm (action)
        self.arm_count[self.last_action]+=1
        
        #Step Size
        step_size=1/self.arm_count[self.last_action]
        
        #Incremental update rule
        self.q_values[self.last_action]=self.q_values[self.last_action]+step_size*(reward-self.q_values[self.last_action])
       
        present_action = our_argmax(self.q_values)

        self.last_action = present_action
        
        return present_action

$\epsilon$-greedy agent¶

Next, let's model an agent that takes the $\epsilon$-greedy approach:

In [7]:
class eGreedyAgent(main_agent.Agent):
    def agent_step(self, reward, observation):
        """
        The function agent_stpe takes in a reward and observation and 
        returns the action the agent chooses at that time step.
        
        Input: 
        reward: the reward the agent recieved from the environment after taking the last action.
        observation: the observed state the agent is in. 
        Returns:
        current_action: the action chosen by the agent at the current time step.
        """
        
        #Action value update
        self.arm_count[self.last_action]+=1
        
        step_size=1/self.arm_count[self.last_action]
        
        #Incremental update rule
        
        self.q_values[self.last_action]=self.q_values[self.last_action]+step_size*(reward-self.q_values[self.last_action])
        
        #Action Selection (e-greedy approach)
        our_prob = np.random.random()
        if our_prob < self.epsilon:
            current_action = np.random.choice(self.num_actions)
        else:          
            current_action = our_argmax(self.q_values)
        
        self.last_action = current_action
        
        return current_action

Averaging multiple runs¶

You may have noticed that we average the results over 250 runs. Why do you think we did that?

To get some insight, let's look at the results of four individual runs by the same agent:

Notice how the four runs are different due to randomness in the environment and the agent. The results will vary depending on the action the agent randomly starts with, or when it randomly chooses to explore. Even if the agent chooses the same action, the reward is randomly sampled from a Gaussian distribution. So there are many sources of variability that it makes sense to average over.
