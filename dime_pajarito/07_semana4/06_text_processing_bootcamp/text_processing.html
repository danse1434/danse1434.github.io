Text Processing¶

What is text processing?¶

Text processing is the practice of manipulating text data in order to make it more amenable to analysis and modeling. There are a whole host of powerful libraries dedicated to this, including:

  • string and str.methods
  • Regular expressions
  • Natural language processing libraries such as nltk. gensim, and spaCy

Why is it important?¶

Text processing is an essential part of performing data analytics or modeling on string data. Unlike numerical and even categorical variables, text data can't be easily structured in a table format and has its own, very unique, and rather complex set of rules that it follows. Engaging in text processing allows us to move onto more complex tasks which are unique to dealing with text, such as:

  • Part-of-speech tagging
  • Named Entity Recognition (NER)
  • Question answering
  • Speech recognition
  • Text-to-speech and speech-to-text
  • Topic modeling
  • Sentiment classification
  • Language modeling
  • Translation

Regular expressions (regex)¶

Regular expressions allow you to use formulas and patterns to work with strings. They are very versatile and are handled via a standard library available in most common programming languages (e.g. Python, SQL, Javascript, etc.). We'll see examples of what this means sooon, but here are some common use cases to begin introducing you to the power of these:

  • Sanitizing a string to ensure that it satisfies a known set of rules (e.g. checking if a given string matches password rules)
  • Filtering or extracting portions of text based on abstract components like letters, numbers, punctuation, etc.
  • Replacing parts of strings based on certain patterns; e.g. only at the start or the end of a string, only whole words, based on surrounding text, etc.

Anchors¶

Anchors are regex tokens that don't match specific characters, but that assert something about the text. For example, anchors can assert that a regular expression should match only in a well-determined location; e.g. the beginning of the string, or the end of the line.

from unicodedata import normalize

def formatingText(text):
	text = text.lower()
	text = re.sub('<.*?>', '', text)
	text = re.sub(':.*?:', '', text)
	text = re.sub(r"([^n\u0300-\u036f]|n(?!\u0303(?![\u0300-\u036f])))[\u0300-\u036f]+", r"\1", normalize( "NFD", text), 0, re.I)
	text = normalize( 'NFC', text)
	text = re.sub('[^a-z ]', '', text)
	return text

gensim¶

gensim is a library for natural language processing created by Radim ehek. Its strong point is theme modeling. That is, it can automatically identify what a set of documents is about. It is also useful for building or importing distributed vector representations, such as word2vec. We can also use gensim to analyze the similarity between documents, which is very useful when we do searches. To learn more, watch the Gensim tutorials.
